# ⏱ Time Awareness — AI 时间差系统

[![License](https://img.shields.io/badge/license-MIT-blue)](LICENSE)
[![Bash](https://img.shields.io/badge/bash-5.0%2B-green)]()

让 AI Agent 拥有「现在几点、用户在不在」的时间感知能力。

**核心理念**：AI 的每轮对话都应该知道当前时间，以及用户在这个时段的行为特征。不用改模型、不用改记忆系统——只加一个薄层。

---

## 快速开始

```bash
bash temporal_context.sh
# 输出: 🕐 北京时间 11:15 CST | 用户活跃度:低
```

将输出注入 AI Agent 的 system prompt 或对话上下文即可。

---

## 为什么需要时间差？

当前大多数 AI Agent 有三个时间盲区：

| 盲区 | 表现 | 根因 |
|------|------|------|
| **时钟盲** | 不知道现在几点，靠上下文语义猜 | 没有主动查时间机制 |
| **用户盲** | 不知道用户在不在活跃时段 | 行为数据未接入 prompt |
| **衰减盲** | 上下文里新旧消息权重相同 | 没有时间戳降权 |

典型翻车：上午 11:00 对用户说「晚安」——上下文里有凌晨的语义模型，但 AI 没查时间。

---

## 架构

```
每次对话前
    │
    ▼
temporal_context.sh（bash 零依赖）
    │  查系统时间 → 映射活跃度时钟
    │  输出: 🕐 HH:MM | 活跃度:等级
    │
    ▼
注入 AI 对话上下文
    │
    ├─ 影响回复模式（简洁/正常/活泼）
    ├─ 辅助串线防御（旧消息自然降权）
    ├─ 数据驱动归寝判断
    └─ 可扩展联动记忆检索权重
```

---

## 活跃度时钟

默认值来自 709 条真实消息的统计，可替换为用户自己的数据：

| 时段 | 活跃度 | 消息占比 | 交互模式 |
|------|--------|----------|----------|
| 06:00–12:00 | 低 | ~12% | 简洁模式 |
| 12:00–16:00 | 中低 | ~15% | 默认模式 |
| 16:00–18:00 | 峰值 | 集中爆发 | 积极参与 |
| 18:00–22:00 | 中高 | ~30% | 默认模式 |
| 22:00–02:00 | 中 | ~25% | 默认模式 |
| 02:00–06:00 | 极低 | ~13% | 简洁 + 可建议休息 |

## 联动设计（嵌入，不修改）

本系统设计为**独立嵌入**现有 AI Agent 架构，不动任何子系统：

```
时间差系统（薄层）
    │  读: 画像数据（活跃时段基准）
    │  输出: 时间感知信号
    │
    ├─→ 记忆系统  加时间条件化检索权重（不改 RAG 索引）
    ├─→ 串线防御  时间距离降权旧消息（不改 context-integrity）
    ├─→ 画像系统  检测活跃时段偏移 → 触发画像刷新
    └─→ SOUL      行为模式变化 → 更新 agent 性格描述
```

### 记忆检索联动

```
凌晨 2:00 检索 → 技术类记忆权重 ×1.5
下午 5:00 检索 → 交互类记忆权重 ×1.2
上午 11:00 → 默认权重
```

实现方式：不改 RAG 索引结构，在检索结果返回后加时间重排序层。

### 串线防御联动

```
消息时间差 > 2h → 旧消息权重 ×0.3（大概率话题已切换）
消息时间差 > 6h → 旧消息权重 ×0.1（几乎肯定是新对话）
```

比纯规则驱动更精准——时间距离是天然的分段信号。

### 双向更新画像

```
时间差输出: "活跃度低"
实际行为: 用户高频回复
         ↓
累积偏差 → 活跃时段可能已偏移 → 触发画像数据刷新
```
